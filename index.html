Web Dev with Git (version control)

Create a remote repository
  • Login to your Github account
  • Select repositories
  • Click "New" on the right side
  • Enter your desired Repository name
  • Click "Create repository"
Clone a remote repository
  • git clone [remote_repository_url] to clone a repository
Initialize local repository (Make sure Git is installed)
  • Go to your project directory
  • git init to initialize your local repository
Git status
  • git status to view all the files which are going to be staged to the commit
Adding files
  • git add [file_name] to add the new/modified single file to the repository
  • git add . to add all the new/modified files to the repository
Commits
  • git commit -m "[your_message]" adding message to the working repository
  • git commit -am "[your_message]" automatically adds the tracked and untracked changes to the repository
Remote repository
  • git remote -v to see available remote
  • git remote add [remote_name] [remote_url] to add a remote
  • git remote remove [remote_name] to remove a remote
Branch
  • git branch to show all available branches in the repository. (The highlighted one is the current branch)
  • git branch [branch_name] to create a branch with a specific branch name
  • git checkout [branch_name] to switch to a specific branch. Alternatively, you can use a single command to create a new branch and switch to the new branch by git checkout -b [branch_name]
Push & Pull
  • git push [remote_name] [branch_name] to push the commit to the remote repository
  • git pull [remote_name] [branch_name] to pull changes from the remote repository
