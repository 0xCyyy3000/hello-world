Web Dev with Git (version control)

Create a remote repository
  • Login to your Github account
  • Select repositories
  • Click "New" on the right side
  • Enter your desired Repository name
  • Click "Create repository"
Clone a remote repository
  • git clone [remote_repository_url] clones a repository
Initialize local repository (Make sure Git is installed)
  • Go to your project directory
  • git init initializes an empty repository or reinitializes the repository
Git status
  • git status displays the state of the working directory and the staging area
Adding files
  • git add [file_name] adds the new/modified single file to the repository
  • git add . adds all the new/modified files to the repository
Commits
  • git commit -m "[your_message]" adds message to the working repository
  • git commit -am "[your_message]" automatically adds the tracked and untracked changes to the repository
Remote repository
  • git remote -v shows all available remote repository
  • git remote add [remote_name] [remote_url] adds a remote repository
  • git remote remove [remote_name] removes a remote repository from the local repository
Branch
  • git branch shows all available branches in the repository. (The highlighted one is the current branch)
  • git branch [branch_name] creates a branch with a specific branch name
  • git branch switch [branch_name] switches current branch to [branch_name] git branch -d [branch_name]with "-d" option to delete the specified branch from the repository.
  • git checkout [branch_name] to switch to a specific branch. Alternatively, you can use a single command to create a new branch and switch to the new branch by git checkout -b [branch_name]
Merge
  • git merge [branch_name] merges the [branch_name] to the current branch
Push & Pull
  • git push [remote_name] [branch_name] pushes the commit to the remote repository
  • git pull [remote_name] [branch_name] pulls changes from the remote repository
